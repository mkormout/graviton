# Summary: Fix Linux export preset platform field

## What was done

Changed `platform="Linux"` → `platform="Linux/X11"` in `export_presets.cfg` preset.1.

## Why it failed

The recent commit `56591c3` renamed `name="Linux/X11"` to `name="Linux"` to match the CI command (`--export-release "Linux"`), which was correct. However, `platform="Linux"` was already wrong for Godot 4.2.1 — the Linux export plugin registers itself as `"Linux/X11"` in 4.2.x. Godot couldn't find the platform plugin, `preset->is_valid()` returned false, and the preset was silently dropped from the registry, producing the "Invalid export preset name: Linux" error.

## Result

- `name="Linux"` — unchanged, matches CI `--export-release "Linux"` 
- `platform="Linux/X11"` — restored, matches Godot 4.2.1's registered plugin name
