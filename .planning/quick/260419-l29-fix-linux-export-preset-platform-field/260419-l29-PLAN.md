# Quick Task 260419-l29: Fix Linux export preset platform field

## Goal

Fix the Linux CI export failure. In Godot 4.2.1, the Linux export plugin is registered as `"Linux/X11"`, not `"Linux"`. The recent commit changed `name="Linux/X11"` → `name="Linux"` (correct, matches the CI command), but also had `platform="Linux"` which causes `preset->is_valid()` to return false — Godot can't find the plugin — so the preset is silently skipped.

## Root Cause

`export_presets.cfg` preset.1:
- `name="Linux"` — correct (CI uses `--export-release "Linux"`)
- `platform="Linux"` — **wrong** for Godot 4.2.1; must be `"Linux/X11"`

## Tasks

### Task 1: Fix platform field in export_presets.cfg

**Files:** `export_presets.cfg`
**Action:** Change `platform="Linux"` to `platform="Linux/X11"` in the Linux preset (preset.1)
**Verify:** `grep -A2 '\[preset.1\]' export_presets.cfg` shows `platform="Linux/X11"`
**Done:** preset is recognized by Godot 4.2.1 and CI Linux build succeeds
