## Root Cause Analysis

The error is a null-receiver call: `mount_front` is `null` at line 35 of `ship-bfg-23-inventory.gd`.

**Call chain that fails:**
1. `ship-bfg-23-inventory.gd:31` -- `var mount_front = ship.get_mount("")`
2. `mountable-body.gd:57-62` -- `get_mount()` iterates `self.mounts` looking for `tag == ""`
3. `mountable-body.gd:15` -- `var mounts = []` (initialized as empty array, never populated at startup)
4. `get_mount()` returns `null` because `mounts` is empty
5. `ship-bfg-23-inventory.gd:35` -- `mount_front.link_slot(...)` crashes on null

**Why `mounts` is empty:**
- `MountableBody` has NO `_ready()` function
- `mounts` is only populated inside `mount_weapon()` via `mounts = get_mounts()` (line 34)
- At startup, no weapons have been mounted yet, so `mounts` remains `[]`
- `get_mounts()` uses `find_children("*", "MountPoint")` which WOULD find the 3 mount point
  nodes (MountPointDefault, MountPointLeft, MountPointRight) -- it just never gets called

**Inheritance chain `_ready()` flow:**
- `PlayerShip` -- no `_ready()`, delegates up
- `Ship._ready()` -- connects signals, calls `super()` (goes to `Body._ready()`)
- `Body._ready()` -- sets `health = max_health` (no further super)
- `MountableBody` -- SKIPPED, has no `_ready()`

**Fix:** Add `_ready()` to `MountableBody` that calls `mounts = get_mounts()`, and ensure
the `super()` chain passes through it correctly.
</root_cause>

**In `components/mountable-body.gd`:**

Add a `_ready()` function that populates the `mounts` array. Place it after the `mounts` var
declaration and before `_physics_process`:

```gdscript
func _ready():
	mounts = get_mounts()
```

This calls `find_children("*", "MountPoint")` which discovers MountPointDefault (tag=""),
MountPointLeft (tag="left"), and MountPointRight (tag="right") from the scene tree.

**In `components/ship.gd`:**

`Ship._ready()` currently calls `super()` which resolves to `Body._ready()` because
`MountableBody` had no `_ready()`. After adding `MountableBody._ready()`, the `super()` in
`Ship._ready()` will now correctly call `MountableBody._ready()`.

However, `MountableBody._ready()` must also chain to `Body._ready()` to preserve the
`health = max_health` initialization. Add `super()` at the end of the new
`MountableBody._ready()`:

```gdscript
func _ready():
	mounts = get_mounts()
	super()
```

**Godot _ready() execution order verification:**
- Children `_ready()` fires before parent `_ready()` in Godot
- `ship-bfg-23-inventory` is a child of `ShipBFG23` (the ship node)
- So the inventory's `_ready()` fires BEFORE the ship's `_ready()`
- This means even with the fix, `mounts` won't be populated yet when the inventory tries to use it

**CRITICAL: This means the Godot child-before-parent _ready() order is the deeper issue.**

To handle this, the inventory script should NOT rely on parent's `_ready()` having already
run. Instead, there are two viable approaches:

**Approach A (preferred -- defensive get_mount):** Make `get_mount()` lazy-initialize `mounts`
if it's empty. In `mountable-body.gd`, change `get_mount()` to:

```gdscript
func get_mount(tag: String = "") -> MountPoint:
	if mounts.is_empty():
		mounts = get_mounts()
	for mount in mounts:
		if mount.tag == tag:
			return mount
	return null
```

Also still add `_ready()` with `mounts = get_mounts()` and `super()` for the normal case
(covers `_physics_process` which also uses `mounts`).

**Approach B (alternative -- defer inventory init):** Use `call_deferred` in the inventory
script. Rejected because it changes the inventory code rather than fixing the core issue in
MountableBody.

**Apply Approach A:**

1. In `components/mountable-body.gd`, add `_ready()`:
```gdscript
func _ready():
	mounts = get_mounts()
	super()
```

2. In `components/mountable-body.gd`, update `get_mount()` to lazy-init:
```gdscript
func get_mount(tag: String = "") -> MountPoint:
	if mounts.is_empty():
		mounts = get_mounts()
	for mount in mounts:
		if mount.tag == tag:
			return mount
	return null
```

3. No changes needed to `components/ship.gd` -- the existing `super()` call in `Ship._ready()`
   will now chain through `MountableBody._ready()` automatically.

**Do NOT change:**
- `ship-bfg-23-inventory.gd` -- the caller is fine, the callee was broken
- `mount-point.gd` -- `link_slot()` is correct
- `body.gd` -- `_ready()` is correct
