  <name>Task 1: Upgrade WaveManager to multi-type waves with auto-countdown</name>
  <files>components/wave-manager.gd</files>
  <action>
Rewrite wave-manager.gd with the following changes. Keep existing signals but update and add:

**Signal changes:**
- Change `wave_started(wave_number: int, enemy_count: int)` to `wave_started(wave_number: int, enemy_count: int, label_text: String)` — label_text is the human-readable announcement string (e.g. "Wave 3 — Hunters + Snipers")
- Add `countdown_tick(seconds_remaining: int)` — emitted once per second during the countdown
- Keep `enemy_count_changed` and `all_waves_complete` unchanged

**Wave data format change:**
- Old: `{ "enemy_scene": PackedScene, "count": int }`
- New: `{ "groups": Array[Dictionary], "label": String }` where each group is `{ "enemy_scene": PackedScene, "count": int }`
- The `"label"` key is the announcement text (e.g. "Hunters + Snipers")
- Keep backwards-compat: if a wave dict has `"enemy_scene"` key (old format), treat it as a single-group wave automatically so existing calls don't crash

**Countdown logic:**
- Add `@export var countdown_seconds: float = 5.0`
- After `_on_wave_complete()` fires (all enemies dead), if more waves remain: start a `Timer` (one-shot, 1s interval) that counts down from `countdown_seconds`. Emit `countdown_tick(seconds_remaining)` each tick. When timer reaches 0, call `trigger_wave()` automatically.
- Do NOT auto-start the very first wave — first wave still requires manual `trigger_wave()` call (F key in world.gd) so the player can orient.
- If `_current_wave_index >= waves.size()` after a wave, do NOT start countdown — emit `all_waves_complete` as now.

**Spawn logic:**
- Update `trigger_wave()` to iterate over all groups in the wave's `"groups"` array, spawning each group's enemies. Total `_enemies_alive` = sum of all group counts.
- Compute `label_text`: use wave's `"label"` key if present, else build from group count as `"Wave N"`.
- Emit `wave_started(_current_wave_index, total_count, label_text)` after incrementing index.

**Timer creation:** Create the countdown Timer in `_ready()` as a child node (one-shot=false, autostart=false), store as `_countdown_timer`. Use a separate `_countdown_remaining: int` var. Connect `_countdown_timer.timeout` to `_on_countdown_tick`. In `_on_countdown_tick`: decrement `_countdown_remaining`, emit `countdown_tick(_countdown_remaining)`, if `_countdown_remaining <= 0` stop timer and call `trigger_wave()`.

Preserve all existing spawn position logic (`_get_spawn_position`), `_spawn_enemy`, `_find_player`, `setup_spawn_parent` call pattern.
