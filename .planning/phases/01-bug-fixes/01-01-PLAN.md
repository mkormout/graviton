Replace the broken `body_entered` function in `components/ship.gd` with the fixed version below.

The broken function (lines 37-45) has two problems:
1. `ray.target_position = body.global_position` — this is WRONG; target_position is local-space when the ray is a child, must use `to_local()`
2. `ray.get_collision_point()` is called before `add_child(ray)` — the ray is never in the tree so the result is always `Vector2.ZERO`
3. `attack.kinetic = 1000` — hardcoded, not speed-scaled

Replace the entire `body_entered` function with:

```gdscript
func body_entered(body):
	var ray = RayCast2D.new()
	ray.target_position = to_local(body.global_position)
	add_child(ray)
	ray.force_raycast_update()
	var contact_point = ray.get_collision_point()
	ray.queue_free()

	var speed = body.linear_velocity.length() if body is RigidBody2D else 0.0
	var attack = Damage.new()
	attack.kinetic = speed / 10.0
	damage(attack)
```

Key implementation rules:
- `to_local(body.global_position)` converts the target to local space so the ray direction is correct when it is a child of the ship node
- `add_child(ray)` MUST happen before `force_raycast_update()` — the physics server only populates raycast results for nodes in the tree
- `ray.queue_free()` after reading — this is a one-shot query, not a persistent node
- `speed / 10.0` gives 100 kinetic damage at 1000 px/s — this is the starting formula per Claude's Discretion; it is a playtesting baseline, not a final value
- Do NOT remove the `contact_point` variable — keep it even if unused in this phase, as it will be needed for Phase 2 hit effects
- Do NOT change any other part of ship.gd

Replace the broken `reload()` function in `components/mountable-weapon.gd` with the fixed version below.

The broken function (lines 67-71) calls `reload_timer.connect("timeout", reloaded)` unconditionally on every invocation. After N reloads, `reloaded()` fires N times per timeout, refilling magazine N× and producing negative ammo counts.

Two fixes are required:

Fix 1 — Guard against starting a reload while one is already in progress (Pitfall 3 from RESEARCH.md: CONNECT_ONE_SHOT still stacks if reload() is called mid-reload):

Fix 2 — Use `CONNECT_ONE_SHOT` so the signal auto-disconnects after one firing.

Replace the entire `reload()` function with:

```gdscript
func reload() -> void:
	if is_reloading():
		return
	reload_timer.start()
	reload_timer.connect("timeout", reloaded, CONNECT_ONE_SHOT)
	if reload_sound:
		reload_sound.play()
```

Key implementation rules:
- `if is_reloading(): return` — `is_reloading()` already exists at line 58 and returns `not reload_timer.is_stopped()`; this guard prevents stacking when reload() is called mid-reload
- `CONNECT_ONE_SHOT` is the Godot 4 built-in `ConnectFlags` enum value; pass it as the third argument to `connect()`
- Do NOT change `reloaded()`, `is_reloading()`, `can_shoot()`, or any other function in this file
- Do NOT change the signal name — it remains `"timeout"` connecting to `reloaded`
