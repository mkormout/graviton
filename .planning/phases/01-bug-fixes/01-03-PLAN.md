Note: Explosion and Item scenes are spawned dynamically; they get spawn_parent via the
parent component (e.g., the Explosion prefab that Body spawns does NOT need its own
spawn_parent wired at the prefab level — the spawned explosion inherits nothing, and its
own generate_debris() call needs spawn_parent). This means the wiring must happen in
world.tscn at the scene level for any Explosion nodes that are permanent children of the
scene, OR the approach is: Body.die() spawns Explosion which then needs spawn_parent set.

Practically: the Explosion scene instantiated by body.die() will have spawn_parent = null
unless the Explosion script is given spawn_parent as part of instantiation OR the explosion
prefab itself has a spawn_parent wired. Since Explosion is instantiated dynamically (not
placed in editor), passing spawn_parent via a property after instantiation is needed.

IMPORTANT implication for the task: After adding spawn_parent to explosion.gd, body.gd's
die() function must also set the explosion's spawn_parent before adding it to the tree:

```gdscript
if death:
    var node = death.instantiate()
    node.global_position = global_position
    if node.has_method("set") and "spawn_parent" in node:
        node.spawn_parent = spawn_parent
    if spawn_parent:
        spawn_parent.add_child(node)
    else:
        push_warning("spawn_parent not set on " + name)
```

Or more simply (since Explosion is a known type):
```gdscript
if death:
    var node = death.instantiate()
    node.global_position = global_position
    node.spawn_parent = spawn_parent  # propagate before adding to tree
    if spawn_parent:
        spawn_parent.add_child(node)
    else:
        push_warning("spawn_parent not set on " + name)
```

This propagation ensures the explosion can spawn its own debris. The executor must
add this propagation line to body.gd:die() as part of Task 1 before the checkpoint.

After Plan 02, `body.gd:die()` looks like this:

```gdscript
func die(delay: float = 0.0):
    if dying:
        return
    if delay:
        await get_tree().create_timer(delay).timeout
    dying = true

    if death:
        var node = death.instantiate()
        node.global_position = global_position
        if spawn_parent:
            spawn_parent.add_child(node)
        else:
            push_warning("spawn_parent not set on " + name)

    if not successors.is_empty():
        for i in range(successors_count):
            add_successor(successors.pick_random(), 200, 2000)

    if item_dropper:
        item_dropper.drop()

    queue_free()
```

The spawned `death` scene is an Explosion, which also has `@export var spawn_parent: Node` (from Plan 02). But since it is instantiated dynamically (not placed in the editor), its `spawn_parent` will be null — meaning `generate_debris()` inside the explosion will push_warning and skip debris.

Fix: propagate `spawn_parent` to the spawned node before adding it to the tree.

In the `if death:` block, add `node.spawn_parent = spawn_parent` after `node.global_position = global_position`:

```gdscript
    if death:
        var node = death.instantiate()
        node.global_position = global_position
        node.spawn_parent = spawn_parent
        if spawn_parent:
            spawn_parent.add_child(node)
        else:
            push_warning("spawn_parent not set on " + name)
```

This works because:
- `node` is the Explosion scene instance, which now has `spawn_parent` as a script property (from Plan 02)
- Setting it before `add_child` means the Explosion's `_ready()` runs with spawn_parent already set
- GDScript property assignment on an instantiated scene is safe before the node is in the tree

Do NOT change anything else in body.gd. This is the only addition needed.
