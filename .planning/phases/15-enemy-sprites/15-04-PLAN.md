    For EACH of the five enemy scenes (beeliner.tscn, sniper.tscn, flanker.tscn, swarmer.tscn, suicider.tscn):

    1. Open the scene in the 2D editor.
    2. Select the root enemy node; look at the Inspector to find the @export var `sprite_region` (inherited from the script).
    3. Click the Rect2 value → click the small "..." / Region Editor button if available, OR click on `$Sprite2D` in the scene tree and temporarily set `region_enabled = true`, `texture = ships_assests.png` to see the region box in the editor.
    4. Use the Region editor to visually select the correct ship sprite for this enemy:
       - Beeliner = ENM-07 (leftmost in top half)
       - Sniper = ENM-08
       - Flanker = ENM-09
       - Swarmer = ENM-10
       - Suicider = ENM-11 (rightmost — disc/sphere with red gem)
    5. Note the exact Rect2 coordinates. If they differ from the Plan 01 defaults, either:
       - Update the @export default constant in `components/{enemy}.gd`, OR
       - Set an explicit scene-level override on the root node in `{enemy}.tscn`

    (Prefer updating the @export default in the .gd so the value is the source of truth and any unplayed enemy instance inherits correctly.)

    ## Step 3 — Verify gem light position

    For EACH enemy scene, after step 2:
    1. Launch the scene (F6) or run the full game briefly (F5) and observe the gem light halo.
    2. If the halo is not centered over the visible gem in the sprite, open the scene and adjust `GemLight.position` (Vector2) in the Inspector to align with the gem.
    3. Re-run and confirm alignment.

    ## Step 4 — Verify sprite scale matches role tier (SPR-05)

    1. Run the full game (F5), spawn all five enemy types (wait for them to spawn naturally, or edit `world.gd` temporarily to spawn one of each).
    2. Visually compare each enemy's size to the player ship:
       - Beeliner, Sniper should look roughly the same size as the player.
       - Flanker should look ~75% of player size.
       - Swarmer, Suicider should look ~50% of player size.
    3. If sizes are off, adjust `sprite_scale` @export defaults in the corresponding `components/{enemy}.gd`.
    4. Revert any world.gd spawn edits before finishing.

    ## Step 5 — Verify bullet sprite regions

    For EACH bullet scene (beeliner-bullet.tscn, sniper-bullet.tscn, flanker-bullet.tscn, swarmer-bullet.tscn):
    1. Open in editor, select `Sprite2D`.
    2. Click the texture region editor ("..." next to region_rect).
    3. Confirm the selected region in the atlas bottom half matches the firing enemy's visual style.
    4. If wrong, adjust `region_rect` and `scale` values in the scene and save.

    ## Step 6 — Verify gem-light pulse character (SPR-04 per D-05)

    1. Spawn all five enemy types (either naturally or via temporary world.gd edit).
    2. Observe each gem pulse for 5-10 seconds:
       - Beeliner: should feel steady and rhythmic (predictable aggression)
       - Sniper: should feel slow and hypnotic (long period, wide swing)
       - Flanker: should feel mid-tempo rhythmic (orbit energy)
       - Swarmer: should feel quick and flickering
       - Suicider: should feel frantic and urgent
    3. If any pulse feels wrong, adjust the three @export defaults in `components/{enemy}.gd`:
       - `gem_energy_min`, `gem_energy_max`, `gem_pulse_half_period`

    ## Step 7 — Verify SPR-03 fallback (atlas missing)

    2. Run the game (F5).
    3. Confirm the game does not crash; enemies spawn with the Polygon2D Shape visible (the old debug shapes — yellow Beeliner, pink Sniper, etc.).
    4. Check the Godot output log: should see no script errors, but may see a brief load() null-return note.
    5. RENAME BACK to `ships_assests.png` immediately after verification.

    ## Step 8 — Verify wave-20 FPS (SPR-04 culling)

    1. Run the game in release or profile mode.
    2. Survive (or cheat via godmode — press G) until wave 20.

    4. If FPS regresses at wave 20:

       - Confirm every enemy scene has a VisibleOnScreenNotifier2D with a rect that covers the sprite.
       - Report FPS measurement to the developer for a follow-up gap plan.
