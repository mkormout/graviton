<task type="auto">
  <name>Task 1: Fix 4 deprecated string-based connect() calls in ship.gd, bullet.gd, and mountable-weapon.gd</name>
  <files>components/ship.gd, components/bullet.gd, components/mountable-weapon.gd</files>
  <read_first>components/ship.gd, components/bullet.gd, components/mountable-weapon.gd</read_first>
  <action>
Fix all string-based `connect("signal_name", callable)` calls to use the modern signal-object syntax `signal_name.connect(callable)`. There are exactly 4 occurrences across 3 files.

**ship.gd** — In `_ready()` (lines 17-18), there is a naming conflict: `body_entered` is both a signal on RigidBody2D AND a method name on Ship. The method must be renamed to avoid ambiguity with the signal reference.

Change:
```gdscript
func _ready():
    connect("body_entered", body_entered)
    picker.connect("body_entered", picker_body_entered)
    super()
```
To:
```gdscript
func _ready():
    body_entered.connect(_on_body_entered)
    picker.body_entered.connect(picker_body_entered)
    super()
```

Also rename the method `body_entered` (line 37) to `_on_body_entered`:
```gdscript
# Was: func body_entered(body):
func _on_body_entered(body):
```
This rename is required because `body_entered` unqualified now refers to the signal, not the method. The `_on_` prefix follows Godot convention for signal handlers.

**bullet.gd** — In `_ready()` (line 9), same pattern but no naming conflict (`collision` is the method name, `body_entered` is the signal):

Change:
```gdscript
func _ready():
    connect("body_entered", collision)
```
To:
```gdscript
func _ready():
    body_entered.connect(collision)
```

**mountable-weapon.gd** — In `reload()` (line 71), the string-based connect with `CONNECT_ONE_SHOT` flag:

Change:
```gdscript
reload_timer.connect("timeout", reloaded, CONNECT_ONE_SHOT)
```
To:
```gdscript
reload_timer.timeout.connect(reloaded, CONNECT_ONE_SHOT)
```
