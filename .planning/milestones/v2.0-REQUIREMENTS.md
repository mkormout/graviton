# Requirements: Graviton

**Defined:** 2026-04-11
**Core Value:** The mount-and-weapon system must work reliably — ships can equip, fire, and swap weapons without bugs or silent failures.

## v2 Requirements (Milestone 2 — Enemies)

### AI Infrastructure

- [x] **ENM-01**: EnemyShip base class defines `State` enum (idling, seeking, lurking, fighting, fleeing, patrolling, evading, escorting) and virtual `_tick_state`, `_enter_state`, `_exit_state` methods for concrete types to override
- [x] **ENM-02**: EnemyShip base class guards all state ticks and fire calls against the `dying` flag so AI does not execute during the death delay
- [x] **ENM-03**: Enemy movement uses `apply_central_force` steering vectors; maximum speed clamped in `_integrate_forces`, never by direct `linear_velocity` assignment
- [x] **ENM-04**: Enemy detection uses `Area2D` with explicit layer/mask bits; mask values documented in code comments referencing the physics layer table in `world.gd`
- [x] **ENM-05**: Enemy fire uses a simplified `Timer`-based fire loop that instantiates a bullet scene at a barrel `Node2D` using `spawn_parent.add_child()` — not `get_tree().current_scene`
- [x] **ENM-06**: Enemy projectiles carry a fixed energy `Damage` resource configured per enemy type (not speed-scaled)

### Enemy Types

- [x] **ENM-07**: Beeliner — seeks player using SEEKING state, transitions to FIGHTING when in range and fires; no picker node in scene
- [x] **ENM-08**: Sniper — maintains standoff distance in SEEKING, fires slow heavy shots in FIGHTING, transitions to FLEEING when player enters close range
- [x] **ENM-09**: Flanker — orbits player in LURKING state using tangential steering force + radius correction before entering FIGHTING; no picker node in scene
- [x] **ENM-10**: Swarmer — low HP, cluster-spawned, reduces thrust when near group members (proximity cohesion without full Boids); no picker node in scene
- [x] **ENM-11**: Suicider — seeks player in SEEKING, triggers existing `Explosion` component on `body_entered` contact; no picker node in scene; no fire logic

### Wave Spawning

- [x] **ENM-12**: `WaveManager` node is a standalone child of the World root, spawns enemy waves with configurable wave composition (enemy type + count)
- [x] **ENM-13**: Wave completion is tracked by a counter decremented on each enemy death signal — not by `get_children()` count — to handle deferred frees correctly
- [x] **ENM-14**: Enemies are spawned with a minimum outer-radius margin from occupied positions to prevent physics-separation launch on the first frame
- [x] **ENM-15**: All enemy scenes omit the picker `Area2D` node (enemies do not collect items)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Flocking / Boids | Deferred — get single-enemy AI solid first |
| NavigationAgent2D pathfinding | No nav mesh in open space; 4.5 regression risk |
| LimboAI plugin | C++ GDExtension, adds external dependency, no benefit over native GDScript state machine |
| MountableWeapon for enemies | Reduces coupling; easier to balance enemy difficulty independently |
| Multiplayer | Not planned |
| Automated test suite (GUT) | User opted for manual playtesting |
