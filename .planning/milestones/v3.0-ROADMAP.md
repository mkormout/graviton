## Phase Details

### Phase 10: Health Pack Foundation
**Goal**: Players can collect health packs dropped by enemies and heal their ship
**Depends on**: Phase 9 (Suicider — completes v2.0)
**Requirements**: ITM-01, ITM-02, ITM-03
**Success Criteria** (what must be TRUE):
  1. Any destroyed enemy has a 10% chance to drop a Health Pack item that appears in the world
  2. The Health Pack has a green cross visual with emissive glow and particle aura, distinguishable from coins and ammo
  3. Flying over the Health Pack causes the player ship's health to increase
**Plans:** 2/1 plans complete
Plans:
- [x] 10-01-PLAN.md — Create health-pack item resource, scene with procedural green cross visuals, and fix heal pickup
- [x] 10-02-PLAN.md — Wire health-pack drops into all 5 enemy CoinDropper nodes

### Phase 11: ScoreManager
**Goal**: Kill scoring, wave multiplier, and combo chain work correctly and are verifiable before any HUD is wired
**Depends on**: Phase 10
**Requirements**: SCR-03, SCR-04, SCR-06, SCR-07, SCR-08
**Success Criteria** (what must be TRUE):
  1. Each enemy type awards a distinct point value when destroyed (verifiable via print output before HUD)
  2. Wave multiplier doubles each wave completed without taking damage (x1->x2->x4->x8->x16 cap), and resets to x1 on any damage
  3. Combo counter increments from the 2nd kill in a chain and expires after 5 seconds of no kills, awarding a bonus score
  4. Each combo kill triggers an audible sound with progressively higher pitch per step in the chain
**Plans:** 2/2 plans complete
Plans:
- [x] 11-01-PLAN.md — Signal infrastructure: died/health_changed on Body, score_value on EnemyShip, wave_completed on WaveManager, autoload registration
- [x] 11-02-PLAN.md — ScoreManager implementation: kill scoring, wave multiplier, combo chain, combo audio, enemy scene wiring

### Phase 12: Score HUD
**Goal**: Players can see their current score, kill count, and wave multiplier on screen at all times
**Depends on**: Phase 11
**Requirements**: SCR-01, SCR-02, SCR-05
**Success Criteria** (what must be TRUE):
  1. The HUD displays a running kill count of destroyed enemy ships, updating immediately on each kill
  2. The HUD displays the player's total accumulated score, updating in real time
  3. The HUD displays the current wave multiplier, updating when it changes (on wave clear or damage taken)
**Plans:** 1/1 plans complete
Plans:
- [x] 12-01-PLAN.md — Create score-hud scene with top-right label layout, signal handlers with score flash and multiplier pulse animations, and world.gd wiring

### Phase 13: Leaderboard
**Goal**: Players can enter their name on death and see a local high score table across sessions
**Depends on**: Phase 12
**Requirements**: SCR-09, SCR-10, SCR-11
**Success Criteria** (what must be TRUE):
  1. On player death, a name-entry overlay appears and accepts free-text keyboard input before showing the leaderboard
  2. The top-10 high scores (name + score) are saved to disk and persist across game restarts
  3. The leaderboard is shown on the death screen, with the current run's entry highlighted if it placed
  4. The last entered player name is pre-filled in the name entry field on the next death
**Plans:** 2/1 plans complete
Plans:
- [x] 13-01-PLAN.md — Create death-screen scene and script with ConfigFile persistence, name entry, and leaderboard display
- [x] 13-02-PLAN.md — Wire death screen in world.gd and verify complete death flow
**UI hint**: yes

### Phase 14: Enemy Balancing + Wave Variety + UI Polish
**Goal**: All enemies feel harder and more distinct, waves are more varied and manually triggered, and HUD information is clearer
**Depends on**: Phase 13
**Requirements**: ENM-16, ENM-17, ENM-18, ENM-19, ENM-20, ENM-21, ENM-22, ENM-23, ENM-24, ENM-25, WAV-01, WAV-02, UI-01, UI-02, UI-03, UI-04
**Success Criteria** (what must be TRUE):
  1. All enemies orient a corner or vertex toward the player during movement and combat, and enemies take noticeably more hits to destroy with longer engagement range
  2. Each enemy type behaves according to its tuned profile: Beeliner moves with randomized pathing, Flanker resumes patrol when far, Swarmer shows fast/slow variance across groups, Sniper strafes while aiming, Suicider charges and explodes with increased area
  3. Waves include more enemies with multi-type compositions, and the next wave does not start until the player presses Enter or F after the wave-cleared label appears
  4. Wave announcement labels are large and readable, including a subtitle listing the enemy types in the incoming wave
  5. The cheat sheet lists all current keyboard shortcuts (including v3.0 additions) and can be toggled on/off with a keyboard shortcut
**Plans:** 4/4 plans complete
Plans:
- [x] 14-01-PLAN.md — Enemy stat buffs (HP x2, range x2, bullet speed x1.4), Polygon2D orientation, suicider explosion buff
- [x] 14-02-PLAN.md — Per-type behavioral tweaks: Beeliner jitter, Sniper strafe, Flanker bug fix, Swarmer speed tier
- [x] 14-03-PLAN.md — Wave flow refactor (manual advance, countdown removal), WaveHud wave-clear label, announcement tween polish
- [x] 14-04-PLAN.md — Controls-hint script + toggle + text update, world.gd wiring (keys, instantiation, speed_tier wave configs)
**UI hint**: yes
