## Milestone Summary

**Key Decisions:**

- MusicManager as autoload (not world node) so it survives scene restart without `get_tree().reload_current_scene()`
- MUS-02: preload catalog instead of DirAccess scan — DirAccess is export-unsafe (missing directory at runtime)
- SPR-04 (gem glow): VisibleOnScreenNotifier2D culling — PointLight2D causes FPS cliff at wave 20 with many enemies on screen
- Restart: ship fully re-instantiated (not in-place reset) to avoid stale signal connections from prior session
- `await get_tree().process_frame` in `_restart_game()` after queue_free calls prevents tree_exiting cascade re-showing wave clear label
- Laser bullet reclassed to CharacterBody2D (move_and_collide for bounce physics) — RigidBody2D cannot intercept collision normals per-frame
- LaserWeapon.fire() must NOT call `super.fire()` — super casts bullet to RigidBody2D which fails for CharacterBody2D laser
- RPG lock uses 60-degree cone (widened from 30 to improve UX); lock_progress drives bracket scale from 1.0→0.1
- WeaponHud uses CanvasLayer z=10; connects to ship via `connect_to_ship()` called from world.gd after each mount
- `_wire_heavy_weapon_shake()` extracted as helper to avoid duplicate `fired_heavy` signal connections on restart

**Issues Resolved:**

- Recoil spin bug: `apply_impulse()` with offset caused torque; fixed with `apply_central_impulse()`
- Laser self-hit: laser bullet was hitting its own shooter before escaping collision shape; fixed with `set_collision_mask_value()` exclusion for one frame
- RPG lock brackets appeared but never shrank: `lock_progress` was not being read by HUD; fixed signal wiring
- Phase 17-02 SUMMARY.md was missing (execution completed but not documented); reconstructed from codebase at milestone close

**Technical Debt Incurred:**

- Phase 16 music UAT not fully verified by human playtesting (3 scenarios pending; requires running the game)
- Quick task stubs (5) created but bodies missing — intent preserved in STATE.md deferred items
- WeaponHud bracket display assumes RPG is in `""` slot (center); multi-RPG lock display would need rework
