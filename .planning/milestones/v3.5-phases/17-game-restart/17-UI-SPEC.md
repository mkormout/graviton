## Typography

All font sizes in the death screen UI:

| Role | Size | Applied To | Method |
|------|------|-----------|--------|
| Display / Title | 64px | "GAME OVER" label (both sections) | `.tscn` `theme_override_font_sizes/font_size` |
| Subtitle | 32px | "ENTER YOUR NAME", "HIGH SCORES" labels | `.tscn` `theme_override_font_sizes/font_size` |
| Table header | 22px | RANK / NAME / SCORE column headers | `.tscn` `theme_override_font_sizes/font_size` |
| Table row / Button | 18px | Score rows, SubmitButton, NameInput | `.tscn` and `add_theme_font_size_override` |
| **Play Again button** | **22px** | New "Play Again" Button | `add_theme_font_size_override("font_size", 22)` |

The Play Again button uses 22px — matches the table header weight, sits visually above row-level (18px) text, and signals it is a primary action without competing with the 64px title. Source: RESEARCH.md code examples.

Font weight: Godot default button font (regular weight). No bold override needed — the button styling (raised surface) provides sufficient visual prominence.

## Color

The death screen establishes a clear palette. The Play Again button must stay within it:

| Role | Value | Usage |
|------|-------|-------|
| Dominant surface | `Color(0, 0, 0, 0.72)` | Full-screen semi-transparent backdrop (Backdrop ColorRect) |
| Background | Black (engine default) | World background behind CanvasLayer |
| Primary text | `Color(1, 1, 1, 1)` — white | All non-highlighted labels; default button label |
| Accent — gold | `Color(1.0, 0.843, 0.0)` | "GAME OVER" title labels; current player's leaderboard row |
| Outline | `Color(0, 0, 0, 1)` — black | `font_outline_color` at `outline_size = 3` on all labels |

**Play Again button color contract:**

- Background: Godot default Button style (inherits engine theme) — no color override. The default Godot button surface (grey) is sufficient to distinguish the button from the label rows above it.
- Text color: No override — inherits Godot default button text color (near-white on default theme). This matches the white label text used throughout the screen.
- Do NOT apply gold accent to the Play Again button. Gold is reserved for: "GAME OVER" title labels and the current-run leaderboard row highlight.
- Do NOT add a custom `StyleBoxFlat` — the plain Godot button border is consistent with the existing SubmitButton in `NameSection/VBox`.

## Node Tree Contract

The button is created entirely in GDScript. No changes to `death-screen.tscn`.

```
LeaderboardSection (Control)
└── VBox (VBoxContainer, separation=8)
    ├── TitleLabel          — "GAME OVER", 64px, gold
    ├── ScoresTitleLabel    — "HIGH SCORES", 32px, white
    ├── HeaderRow           — HBoxContainer (RANK / NAME / SCORE at 22px)
    ├── Separator           — HSeparator
    ├── RowsContainer       — VBoxContainer (score rows, 18px)
    └── [NEW] PlayAgainBtn  — Button, "Play Again", 22px  ← added in _on_submit()
```

**Insertion point:** `$LeaderboardSection/VBox.add_child(play_again_btn)` — appended last after `_populate_table()` completes.

**Guard:** Button creation is inside `_on_submit()` which is already guarded by `if _submitted: return`. The button can only be added once per death cycle. No additional guard needed.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Button press | Emits `play_again_requested` signal from `DeathScreen`. No visual feedback beyond standard Godot button press state (hover/active built-in). |
| Signal receiver | `world.gd` connects `death_screen.play_again_requested` to `_restart_game()` in `_ready()`. |
| Button availability | Button appears immediately after `_populate_table()` — no delay, no animation. |
| Button visibility | Button is a permanent child of `$LeaderboardSection/VBox` once added. `LeaderboardSection` is hidden at next death (`show_death_screen()` resets visibility); the button is destroyed only if the VBox is rebuilt — which it is not. The `_submitted` guard ensures a second death does not add a duplicate button to the VBox. |
| Post-click | `DeathScreen.visible = false` is set by `world.gd._restart_game()`. The button does not hide itself. |
