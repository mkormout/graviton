# Phase 13 — UI Design Contract

> Visual and interaction contract for the Leaderboard death screen.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Godot 4 built-in UI nodes only |
| Preset | not applicable |
| Component library | Godot Control nodes (Label, LineEdit, VBoxContainer, HBoxContainer, Button) |
| Icon library | none |
| Font | Godot default (no custom font loaded in project) |

**Note:** This is a Godot 4 GDScript project. No web framework, no shadcn, no npm.
Design is expressed through `.tscn` node properties and `theme_override_*` GDScript calls.

## Spacing Scale

Godot does not use CSS tokens. Spacing is expressed as pixel values in `offset_*`, `separation`,
`margin_*`, and `custom_minimum_size` properties. The following values are the declared contract
(all multiples of 4):

| Token | px Value | Usage in this Phase |
|-------|----------|---------------------|
| xs | 4 | `theme_override_constants/separation` inside HBoxContainer rows |
| sm | 8 | Separation between label columns in leaderboard rows |
| md | 16 | Outer margin of the overlay VBox from screen edge; gap between name entry section and leaderboard section |
| lg | 24 | Separation between "GAME OVER" title and the name entry field |
| xl | 32 | Vertical separation between leaderboard section title ("HIGH SCORES") and the table rows |
| 2xl | 48 | Top offset of the entire overlay panel from viewport center (upward shift to avoid bottom-heavy layout) |

**Source:** Established by score-hud (separation = 4) and convention from existing CanvasLayer layouts.

Exceptions:
- Column widths for the three leaderboard columns are declared in the Typography section below, not in this scale.

## Typography

All text is rendered via Godot `Label` nodes. Font sizes are set via `theme_override_font_sizes/font_size`.
Font weights are not individually configurable with Godot's default font — the two effective weights
come from size differentiation only. Outline: `theme_override_constants/outline_size = 3`,
`theme_override_colors/font_outline_color = Color(0, 0, 0, 1)` on all labels.

| Role | Size (px) | Weight / Style | Line Height | Usage |
|------|-----------|----------------|-------------|-------|
| Display | 64 | Bold (Godot default bold) | 1.2 | "GAME OVER" title label |
| Heading | 32 | Regular | 1.2 | "HIGH SCORES" section title; sub-headings |
| Body | 22 | Regular | 1.5 | Leaderboard column headers (RANK / NAME / SCORE) |
| Label | 18 | Regular | 1.5 | Leaderboard data rows; LineEdit font size |

**Source:** 18px matches the established score-hud label size (verified: score-hud.tscn `font_size = 18`).
22px matches wave-hud panel labels (verified: wave-hud.tscn `font_size = 22`).
64px and 32px are Claude's discretion (CONTEXT.md §Claude's Discretion — exact font sizes).

Leaderboard column minimum widths (set on the first `Label` child of each `HBoxContainer` row):
- Rank column: `custom_minimum_size = Vector2(48, 0)` — accommodates "»10" (4 chars + marker)
- Name column: `custom_minimum_size = Vector2(224, 0)` — accommodates 16-char name at 14px average char width
- Score column: `custom_minimum_size = Vector2(128, 0)` — right-aligned; accommodates 7-digit score

**Source:** Column widths are Claude's discretion (CONTEXT.md §Claude's Discretion).

## Color

Godot uses `Color(r, g, b, a)` values, not hex. Equivalents provided for reference.

| Role | Godot Value | Hex Equiv. | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `Color(0, 0, 0, 0.72)` | `#000000B8` | Semi-transparent black background rect behind entire overlay (ColorRect, 72% alpha). Dims the game scene without fully obscuring it. |
| Secondary (30%) | `Color(0.15, 0.15, 0.15, 0.0)` | transparent | No panel background — bare labels only. Consistent with score-hud and wave-hud pattern (D-11). |
| Accent (10%) | `Color(1.0, 0.843, 0.0)` | `#FFD700` | Highlighted leaderboard row (current run); "GAME OVER" title label; Submit button border/modulate on focus |
| Inactive text | `Color(0.5, 0.5, 0.5, 1)` | `#808080` | Placeholder text in LineEdit before input; combo-inactive rows (inherited pattern from score-hud) |
| Destructive | n/a | n/a | No destructive actions in this phase |

**Source:**
- Gold `Color(1.0, 0.843, 0.0)` — verified in `components/score-manager.gd` line 127 and `prefabs/ui/score-hud.gd` line 75 (D-09).
- White `Color(1, 1, 1, 1)` with black outline — verified in `score-hud.tscn` (all row labels).
- Semi-transparent overlay backdrop: Claude's discretion — 72% black provides legibility without hiding the death scene.

Accent reserved for:
- Current run's leaderboard row (all three column labels)
- The unranked 11th row when not in top-10 (D-10)
- "GAME OVER" display title
- Submit button text when the field is focused (focus indication)

White text `Color(1, 1, 1, 1)` with `outline_size = 3` black outline: all non-highlighted labels.

## Layout Contract

### Stage 1 — Name Entry (visible immediately on player death)

```
[ColorRect — semi-transparent black backdrop, fullscreen]
[CanvasLayer — layer = 20, process_mode = PROCESS_MODE_ALWAYS]
  [VBoxContainer — centered, anchor = 0.5/0.5, grow = both]
    [Label "GAME OVER"] — Display 64px, gold Color(1.0, 0.843, 0.0), centered
    [spacer — 24px]
    [Label "ENTER YOUR NAME"] — Heading 32px, white, centered
    [spacer — 16px]
    [LineEdit] — 18px, max_length = 16, pre-filled with last_name, grab_focus deferred
    [spacer — 16px]
    [Button "SAVE SCORE"] — 18px, white text; on pressed → _on_submit()
```

### Stage 2 — Leaderboard (replaces name entry section after submit)

```
[Same CanvasLayer]
  [VBoxContainer — centered]
    [Label "GAME OVER"] — Display 64px, gold, centered (persists)
    [spacer — 16px]
    [Label "HIGH SCORES"] — Heading 32px, white, centered
    [spacer — 32px]
    [HBoxContainer — column headers]
      [Label "RANK" — Body 22px, white, min-width 48px]
      [Label "NAME" — Body 22px, white, min-width 224px]
      [Label "SCORE" — Body 22px, white, min-width 128px, right-aligned]
    [HSeparator — 2px, Color(0.3, 0.3, 0.3, 1)]
    [VBoxContainer "_rows_container" — rows injected dynamically]
      [HBoxContainer per entry — Label 18px each]
```

Row generation rules (from D-08 through D-10):
- Rows 1–10: rank number, name, score — white unless current run
- Current run's row: `»N` rank marker, gold text on all three labels
- If not in top 10: separator row `[HSeparator]`, then unranked `»–` row still gold, score shown

**Source:** Layout is Claude's discretion (CONTEXT.md §Claude's Discretion — layout choice and spacing).
Row structure verified against score-hud `HBoxContainer` pattern.

## Interaction Contract

### Name Entry

| Trigger | Behavior |
|---------|----------|
| Death overlay appears | `visible = true`, name section shown, leaderboard section hidden, LineEdit pre-filled with `last_name`, `call_deferred("grab_focus")` on LineEdit |
| Player types | Standard LineEdit behavior; max 16 chars enforced by `max_length = 16` |
| Player presses Enter | Calls `_on_submit()` |
| Player clicks Save Score button | Calls `_on_submit()` |
| `_on_submit()` fires | Guard check `_submitted`; if already submitted, return. Set `_submitted = true`. Save score + name to ConfigFile. Transition to leaderboard stage. |
| Name is blank on submit | Save as `"---"` (D-05) |

### Leaderboard Display

| Trigger | Behavior |
|---------|----------|
| Leaderboard stage activated | `_name_section.visible = false`, `_leaderboard_section.visible = true`, rows populated from ConfigFile |
| Current run placed in top 10 | That row rendered gold with `»N` rank prefix |
| Current run not in top 10 | Rows 1–10 rendered normally, separator added, 11th row added gold with `»–` rank |
| No input expected | No keyboard shortcuts; overlay is terminal (no restart in this phase per Deferred) |

### Pause State

- `get_tree().paused = true` called by the death handler before showing overlay (D-01)
- `process_mode = PROCESS_MODE_ALWAYS` set on the **CanvasLayer root node** (not on children) — verified critical requirement from RESEARCH.md Pitfall 1
- All child nodes inherit `PROCESS_MODE_ALWAYS` from the root; no additional per-node setting required unless input issues arise

## Copywriting Contract

| Element | Copy |
|---------|------|
| Death overlay title | `GAME OVER` |
| Name entry prompt | `ENTER YOUR NAME` |
| Name input placeholder | `(16 chars max)` |
| Submit button | `SAVE SCORE` |
| Leaderboard section title | `HIGH SCORES` |
| Column header — rank | `RANK` |
| Column header — name | `NAME` |
| Column header — score | `SCORE` |
| Anonymous entry name | `---` (three dashes — used when name submitted blank, D-05) |
| Current run rank marker | `»N` (guillemet + rank number, e.g. `»3`) |
| Unranked run marker | `»–` (guillemet + em-dash) |
| Empty leaderboard state | Leaderboard table is not shown if no entries exist; only "GAME OVER" + name entry displayed. First submission creates the table. No explicit "no scores yet" copy — the table simply has 0 rows before first submit then shows 1 entry after. |
| Error state | No network calls; no error states possible. ConfigFile write failure is silent — no user-facing error copy needed. |
| Destructive confirmation | None — no destructive actions in this phase (D per Deferred: no delete, no reset) |

**Source:** Title copy `GAME OVER` and `HIGH SCORES` from D-12. All other copy is Claude's discretion.

## Animation Contract

| Element | Animation |
|---------|-----------|
| Overlay appearance | `visible = true` instantaneously — no fade-in (consistent with score-hud and wave-hud approach; no `modulate.a` tween on root) |
| Stage transition (name → leaderboard) | Instantaneous swap: `_name_section.visible = false`, `_leaderboard_section.visible = true` (D-07: "no saving... state") |
| Current run row | No animation; gold color is static |
| Submit button hover | Default Godot Control focus/hover behavior; no custom tween |

**Timers during pause:** If any tween is needed (future enhancement), use `Tween` created via `create_tween()` — Tweens on `PROCESS_MODE_ALWAYS` nodes process correctly while paused. Do NOT use `get_tree().create_timer()` without `process_always = true`.

## Node Structure Contract

```
prefabs/ui/death-screen.tscn
├── DeathScreen : CanvasLayer
│   layer = 20
│   process_mode = 3  (PROCESS_MODE_ALWAYS)
│   visible = false
│   script = death-screen.gd
│   ├── Backdrop : ColorRect
│   │   anchors_preset = 15 (full rect)
│   │   color = Color(0, 0, 0, 0.72)
│   ├── NameSection : Control
│   │   anchors_preset = 8 (center)
│   │   ├── VBox : VBoxContainer
│   │   │   ├── TitleLabel : Label        (GAME OVER — 64px, gold)
│   │   │   ├── PromptLabel : Label       (ENTER YOUR NAME — 32px, white)
│   │   │   ├── NameInput : LineEdit      (18px, max_length=16)
│   │   │   └── SubmitButton : Button     (SAVE SCORE — 18px)
│   └── LeaderboardSection : Control
│       visible = false
│       anchors_preset = 8 (center)
│       └── VBox : VBoxContainer
│           ├── TitleLabel : Label        (GAME OVER — 64px, gold)
│           ├── ScoresTitleLabel : Label  (HIGH SCORES — 32px, white)
│           ├── HeaderRow : HBoxContainer
│           │   ├── RankHeader : Label    (RANK — 22px, white, min 48px)
│           │   ├── NameHeader : Label    (NAME — 22px, white, min 224px)
│           │   └── ScoreHeader : Label   (SCORE — 22px, white, min 128px)
│           ├── Separator : HSeparator
│           └── RowsContainer : VBoxContainer   (rows injected in _ready / _populate_table)
```

**Source:** Structure is Claude's discretion. Follows CanvasLayer + bare-label pattern from D-11 and RESEARCH.md Pattern 3.
