# Phase 14 — UI Design Contract

> Visual and interaction contract for Phase 14: Enemy Balancing + Wave Variety + UI Polish.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Note: This is a Godot 4 GDScript project. There is no web framework, no CSS, no shadcn.
> All design tokens map directly to Godot node properties (`theme_override_font_sizes`,
> `theme_override_colors`, `theme_override_constants`, CanvasLayer `layer`, modulate, anchors).
> The "spacing scale" maps to node offset/margin values. The "color palette" maps to
> `theme_override_colors/font_color` and `modulate` values. Registry safety is not applicable.

## Spacing Scale

Godot uses pixel offsets, not a CSS spacing system. The following values are the in-use and mandated pixel values for this phase, expressed as margin/offset constants.

| Token | Value | Godot Property | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `theme_override_constants/margin_*` | Icon gaps, label inner padding |
| sm | 8px | VBoxContainer `separation`, offset adjustments | Compact row spacing |
| md | 16px | Panel/VBox offset from screen edge | Default element spacing (score-hud top offset: 16px) |
| lg | 24px | — | Section breaks inside panels |
| xl | 32px | — | Inter-panel gap |
| 2xl | 48px | — | Not actively used in this phase |
| 3xl | 64px | — | Not actively used in this phase |

Exceptions:
- `WaveClearLabel`: offset_left = -400, offset_right = 400, offset_top = -60, offset_bottom = 60 (centered overlay; fixed logical size, not a spacing token)
- `AnnouncementLabel`: identical anchor/offset pattern to WaveClearLabel (same centering convention)
- `► ToggleButton`: offset_left = -35 from right edge (flush-to-edge affordance, not spacing)
- `controls-hint MarginContainer`: 5px on all four sides (xs) — established in existing scene

Source: RESEARCH.md verified codebase values

## Typography

This phase introduces or modifies 4 text element roles. All use Godot's default font. Font sizes are set via `theme_override_font_sizes/font_size`.

| Role | Size | Weight | Line Height | Godot Property | Element |
|------|------|--------|-------------|----------------|---------|
| Body | 18px | regular | 1.0 (single-line labels) | `font_size = 18` | ScoreHud rows, leaderboard rows, controls-hint text |
| Label | 22px | regular | 1.0 | `font_size = 22` | WaveHud WaveLabel ("WAVE N") |
| Heading | 48px | regular | 1.2 (multiline) | `font_size = 48` | WaveClearLabel ("WAVE N CLEARED / Press Enter or F to continue") |
| Display | 72px | regular | 1.2 (multiline) | `font_size = 72` | AnnouncementLabel — increased from 48px per D-18 |

Notes:
- Godot's default Label has no weight override; "regular" is the only weight available without a custom font.
- `font_size = 20` is used in existing controls-hint RichTextLabel (body size for that panel). Unchanged.
- Black outline (3px) applied to all HUD text via `theme_override_constants/outline_size = 3` and `theme_override_colors/font_outline_color = Color(0, 0, 0, 1)`. This is the established pattern from score-hud.tscn. Apply to WaveClearLabel and all new labels.

Source: RESEARCH.md (D-18: AnnouncementLabel → 72px), codebase inspection (score-hud.tscn, wave-hud.tscn, death-screen.gd)

## Color

Godot 4 HUD uses no hex CSS colors — all values are `Color(r, g, b, a)` floats. The 60/30/10 model maps as follows for this game context.

| Role | Godot Color | Usage |
|------|-------------|-------|
| Dominant (60%) | `Color(0, 0, 0, 1)` — black | Background (ColorRect 100,000×100,000), viewport background |
| Secondary (30%) | `Color(0.2, 0.2, 0.2, 0.8)` approx | Panel backgrounds (WaveHud Panel, controls-hint Panel) |
| Accent (10%) | `Color(1, 1, 1, 1)` — white | All active HUD label text; wave-clear label text; announcement text |
| Highlight | `Color(1.0, 0.843, 0.0, 1)` — gold | Leaderboard current-run row highlight, multiplier pulse animation |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for:
- WaveClearLabel text (persistent overlay label)
- AnnouncementLabel text (fade-in/hold/fade-out overlay)
- ToggleButton text character (`►` / `◄`)
- Active HUD value labels (score, kills, multiplier, combo)

Inactive/dim state: `Color(0.5, 0.5, 0.5, 1)` — used for combo value when no active combo. Established in score-hud.gd.

Transparency: `modulate.a = 0.0` for hidden; `modulate.a = 1.0` for visible. Used in fade tweens. No semi-transparent text for main labels.

Source: codebase inspection (score-hud.tscn, death-screen.gd, wave-hud.gd); RESEARCH.md pattern notes

## Interaction Contract

This section captures the behavioral contracts for every interactive UI element in this phase. (Godot-specific — replaces the generic registry safety section for non-web stacks.)

### WaveClearLabel

| State | Visible | Text | Alpha |
|-------|---------|------|-------|
| Wave in progress | false | — | 0.0 |
| `wave_cleared_waiting` signal received | true | "WAVE N CLEARED\nPress Enter or F to continue" | 1.0 (no fade — persistent) |
| Player presses Enter or F | false (call `hide_wave_clear_label()`) | — | 0.0 |

- No tween on this label — it appears and disappears instantly (snap, not fade). Persistence is the UX.
- Font size: 48px. Anchor: center screen. Layer: WaveHud CanvasLayer layer 10.

Source: CONTEXT.md D-01, D-03; RESEARCH.md Pattern 6

### AnnouncementLabel (font resize + new tween)

| Phase | Duration | Modulate alpha |
|-------|----------|----------------|
| Fade in | 0.3s | 0.0 → 1.0 |
| Hold | 2.0s | 1.0 (tween_interval) |
| Fade out | 1.0s | 1.0 → 0.0 |

- Replace existing 3s linear fade with this 3-step tween sequence.
- Kill any running tween on `_announcement_label` before starting new one (use stored `_announce_tween` var).
- Font size: 72px (was 48px). No other content or anchor changes.

Source: CONTEXT.md D-18, D-19; RESEARCH.md Pattern 7

### ControlsHint Toggle Button (`►` / `◄`)

| State | Panel visible | Button text |
|-------|--------------|-------------|
| Default (game start) | false | `►` |
| After first TAB or click | true | `◄` |
| After second TAB or click | false | `►` |

- Button type: Godot `Button` node, flat style (no background panel behind the arrow).
- Position: anchored to right edge, vertically centered. `anchor_left = 1.0`, `anchor_right = 1.0`, `anchor_top = 0.5`, `anchor_bottom = 0.5`, `offset_left = -35`.
- The button persists on screen at all times (not hidden with the panel).
- Both `KEY_TAB` in `world.gd _input()` and `Button.pressed` signal call the same `ControlsHint.toggle()` method.

Source: CONTEXT.md D-04, D-05, D-06; RESEARCH.md Pattern 8, Arrow Button Position section

### KEY_ENTER Reassignment

| Condition | Action |
|-----------|--------|
| `_wave_clear_pending == true` | Call `WaveManager.trigger_wave()`, set `_wave_clear_pending = false`, call `wave_hud.hide_wave_clear_label()` |
| `_wave_clear_pending == false` | Do nothing (asteroid spawn removed) |

- Same gate applied to KEY_F (which already triggers wave; now also hides label and clears flag).
- `_wave_clear_pending` is a `bool` instance variable in `world.gd`.

Source: CONTEXT.md D-01, D-02; RESEARCH.md Pattern 9

### HUD Layer Order

| CanvasLayer | Layer | Notes |
|-------------|-------|-------|
| ScoreHud | 10 | Top-right corner — score, kills, mult, combo |
| WaveHud | 10 | Top-center panel + centered overlays (WaveClearLabel, AnnouncementLabel) |
| ControlsHint | default (1) | Right edge panel + toggle button |
| DeathScreen | (higher, existing) | Full-screen overlay on death |

If WaveClearLabel z-fights with DeathScreen, bump WaveHud layer to 11.

Source: RESEARCH.md Pattern 6 layer note, codebase wave-hud.tscn and score-hud.tscn
