**flanker.gd — Fix patrol resumption bug (per D-15, RESEARCH Pattern 3):**

The current `_on_detection_area_body_exited` (lines 46-49) unconditionally clears `_target` and goes IDLING whenever the player exits the DetectionArea radius. This causes Flankers in LURKING or FIGHTING state to freeze.

Replace the current method:
```gdscript
func _on_detection_area_body_exited(body: Node2D) -> void:
    if body == _target:
        _target = null
        _change_state(State.IDLING)
```

With this fixed version (mirrors the correct Sniper pattern from sniper.gd line 36-41):
```gdscript
func _on_detection_area_body_exited(body: Node2D) -> void:
    # Only drop target in SEEKING — LURKING and FIGHTING have their own
    # distance-based leash logic in _tick_state (max_follow_distance check).
    if body == _target and current_state == State.SEEKING:
        _target = null
        _change_state(State.IDLING)
```

This is the ONLY change to flanker.gd. No stat changes here (stats are .tscn scene overrides in Plan 01).

**swarmer.gd — Add speed_tier export (per D-16, RESEARCH Pattern 4):**

1. Add new export after the existing `@export var bullet_speed` line (line 9):
```gdscript
@export var speed_tier: float = 1.0
```

2. Modify `_ready()` to apply `speed_tier` BEFORE the existing `randf_range` variance. The current `_ready()` is:
```gdscript
func _ready() -> void:
    super()
    thrust *= randf_range(0.8, 1.2)
    max_speed *= randf_range(0.8, 1.2)
    _angle_offset = deg_to_rad(randf_range(-40.0, 40.0))
    ...
```

Change it to:
```gdscript
func _ready() -> void:
    super()
    thrust *= speed_tier
    max_speed *= speed_tier
    thrust *= randf_range(0.8, 1.2)
    max_speed *= randf_range(0.8, 1.2)
    _angle_offset = deg_to_rad(randf_range(-40.0, 40.0))
    ...
```

The `speed_tier` multiplication MUST come before the `randf_range` lines. The tier scales the base values, then per-instance variance jitters on top. The rest of `_ready()` (lines after `_angle_offset`) remains unchanged.

IMPORTANT: `speed_tier` will be set by WaveManager on each instance BEFORE `add_child()` in Plan 04 (world.gd wiring). When `speed_tier` is not set (legacy waves), it defaults to 1.0, so existing behavior is unchanged.
