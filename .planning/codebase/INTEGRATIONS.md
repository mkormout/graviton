## CI/CD & Deployment

**CI Pipeline:**
- GitHub Actions (`.github/workflows/gitlab-ci.yml`)
- Trigger: every push to any branch
- Runner: `ubuntu-22.04`
- Container: `barichello/godot-ci:4.2.1` (pre-built Docker image with Godot + export templates)

**Export Jobs (run in parallel):**
- `export-windows` → `build/windows/graviton.exe` (x86_64)
- `export-linux` → `build/linux/graviton.x86_64` (x86_64)
- `export-web` → `build/web/index.html` (HTML5/WebAssembly)
- `export-mac` → `build/mac/graviton.zip` (universal binary)

**Deployment:**
- Web build auto-deployed to GitHub Pages on every push
  - Action: `JamesIves/github-pages-deploy-action@v4`
  - Target repo: `mkormout/graviton-app` (separate repository)
  - Branch: `main`
  - Auth: `secrets.PAT` (Personal Access Token)
- Desktop binaries published as GitHub Release (`tag_name: latest`)
  - Action: `softprops/action-gh-release@v1`
  - Auth: `secrets.GITHUB_TOKEN`
  - Artifacts: Windows `.exe`, Linux `.x86_64`, macOS `.zip`

**GitHub Actions Used:**
- `actions/checkout@v2` / `@v4` - Source checkout (with Git LFS support)
- `actions/upload-artifact@v4` - Store build artifacts between jobs
- `actions/download-artifact@v4` - Retrieve artifacts for release/deploy
- `JamesIves/github-pages-deploy-action@v4` - GitHub Pages deployment
- `softprops/action-gh-release@v1` - GitHub Release creation

## Asset Pipeline

**Images:**
- PNG files in `images/` - Sprite sheets and UI textures
- SVG files (`icon.svg`, `assets.svg`) - Project icon and asset sources
- All images have `.import` sidecar files generated by the Godot editor

**Audio:**
- WAV and MP3 files in `sounds/` - Sound effects for weapons, explosions, pickups
- All audio files have `.import` sidecar files

**Scenes:**
- `.tscn` (text scene) files throughout `prefabs/` - Reusable prefab scenes
- `.tres` (text resource) files in `items/` - Weapon and ammo item definitions

**Texture Compression (export-time):**
- Desktop: BPTC, S3TC
- Mobile/Web: ETC, ETC2, ASTC

## Environment Configuration

**Required secrets (CI only):**
- `secrets.PAT` - GitHub Personal Access Token for cross-repo Pages deployment
- `secrets.GITHUB_TOKEN` - Auto-provided by GitHub Actions for release creation

**No runtime environment variables required** - the game is a standalone binary.
